# 07C_CODEX_WORKFLOW

## Before any task

- Read `AGENTS.md`.
- Read `docs/00_HANDOFF.md`.
- Read `docs/00_DOCS_INDEX.md`.
- Read `docs/09_governance/09_STAGE_TASK_CONTRACT.md`.
- Read `docs/10_next/10_NEXT.md`.

## Rule-load test

During the first rule-load test, do not modify files. Only summarize rules, current stage, forbidden actions and the first next task.

## While coding

Coding is allowed only when the current stage and `docs/10_next/10_NEXT.md` explicitly allow it.

- Change only files needed for the current task.
- Run tests if available.
- Do not add unrequested features.
- Do not add unauthorized platform automation or evasion logic.
- Document evidence, risks and results.

## After completing a task

- Update `docs/10_next/10_NEXT.md`.
- Update `docs/09_governance/09_CHANGELOG.md`.
- Update `docs/00_HANDOFF.md`.
- Record evidence in `docs/09_governance/09_EVIDENCE_LOG.md` when evidence is required.
- Record risks in `docs/09_governance/09_RISK_REGISTER.md` when risk is discovered.
